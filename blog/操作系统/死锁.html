    3/28/2022 死锁

    # 死锁概念

    # 引起死锁的原因

    1. 竞争不可抢占性资源
    2. 竞争可消耗资源
    3. 进程推进顺序不当

    # 死锁产生的四个条件（有一个条件不成立，则不会产生死锁）

    1. 互斥条件：一个资源一次只能被一个进程使用
    2. 请求与保持条件：一个进程因请求资源而阻塞时，对已获得资源保持不放
    3. 不可剥夺条件：进程获得的资源，在未完全使用完之前，不能强行剥夺
    4. 循环等待条件：若干进程之间形成一种头尾相接的环形等待资源关系

    # 处理死锁的方法

    # 预防死锁

    破坏四个条件之一

    1. 破坏不可抢占性：进程已经占有的资源被短暂地释放
    2. 破坏请求和保持条件：一个进程在请求资源时，不能持有不可抢占性资源。
    3. 破坏循环等待条件：对系统的所有资源进行线性排序，并且赋予不同的序号。规定每个进程必须按照序号递增的顺序请求资源。每个线程以相同的顺序申请自己想要的资源。

    # 避免死锁

    在资源动态分配的过程中，防止系统进入不安全状态。银行家算法

    • 安全序列：按照某种进程推进顺序为每个进程分配其所需要的资源，直至满足每个进程对资源的最大需求，使得每个进程都可以顺利地完成。
    • 可利用资源向量：
    • 最大需求矩阵
    • 分配矩阵：已分配的资源数
    • 需求矩阵
    • 请求向量
    • 工作向量：系统可以提供给进程继续运行所需要的资源数目。初始化为Avaliable向量
    • Finish：系统是否有足够的资源分配给进程

    # 检测死锁

    1. 保存有关资源的请求和分配信息
    2. 提供算法检测是否发生死锁

    # 解除死锁

    1. 抢占资源：从一个或者多个进程中抢占足够数量的资源，分配给死锁进程，解除死锁状态
    2. 终止进程：终止一个或者多个进程，直至打破循环环路，使得系统从死锁状态中解脱出来。
